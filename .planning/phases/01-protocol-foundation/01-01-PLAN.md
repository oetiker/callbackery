Create all core schemas following RFC 9457 and CONTEXT.md decisions:

**User.yaml:**
- id (string, uuid), email (string, email), name (string), createdAt (date-time)
- Required: id, email, name
- Include realistic examples

**Plugin.yaml:**
- id (string), type (enum: form, table, action, html, cardlist), name (string), title (string)
- Required: id, type, name
- Example: user management form plugin

**PluginConfig.yaml:**
- type (enum: form, table, action, html, cardlist)
- fields (array of field definitions) - for forms
- columns (array of column definitions) - for tables
- actions (array of action definitions)
- Use oneOf/discriminator pattern based on type

**Navigation.yaml and NavigationItem.yaml:**
- NavigationItem: id, label, pluginId (optional), children (recursive array of NavigationItem)
- Navigation: wrapper with items array
- Supports hierarchical menu structure

**Session.yaml:**
- token (string), userId (string, uuid), expiresAt (date-time)
- For authentication response

**ProblemDetails.yaml (RFC 9457):**
- type (uri), title (string), status (integer), detail (string), instance (uri)
- errors (array of field errors) - extension for validation
- Required: type, title, status
- Content-Type: application/problem+json

**ValidationError.yaml (field-level):**
- field (string), code (string enum: REQUIRED, INVALID_FORMAT, OUT_OF_RANGE, etc.), message (string)
- Required: field, code, message

**ResponseEnvelope.yaml:**
- Generic wrapper: data (any), meta (object with version, timestamp)
- Required: data

**PaginationMeta.yaml:**
- total (integer), offset (integer), limit (integer)
- For list responses

**Reusable responses (spec/components/responses/):**
- ValidationError.yaml: 422 with ProblemDetails + errors array
- NotFound.yaml: 404 with ProblemDetails
- Unauthorized.yaml: 401 with ProblemDetails

**Reusable parameters:**
- PluginId.yaml: path parameter for plugin ID

Update _index.yaml files to reference all schemas, responses, parameters.

All schemas must have:
- Meaningful descriptions
- Realistic examples (not "string" or 123)
- Proper type declarations (explicit, not implicit)
